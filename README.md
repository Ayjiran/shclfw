## 🚀 快速开始

### 环境要求

- PHP >= 7.4
- MySQL >= 5.7
- Apache/Nginx Web服务器

### 安装部署

1. **上传文件**  
   将系统文件上传至网站根目录

2. **配置数据库**  
   修改 `db.php` 文件中的数据库连接信息：
   ```php
   $host = 'localhost';      // 数据库地址
   $dbname = 'aftersales';   // 数据库名
   $username = 'root';       // 用户名
   $password = '';           // 密码
   ```

3. **自动安装**  
   首次访问系统会自动创建数据库和表结构

4. **访问系统**  
   - 前台地址: `http://您的域名/index.php`
   - 后台地址: `http://您的域名/admin/login.php`

## 👥 前台用户使用指南

### 提交售后申请

1. 访问前台首页
2. 选择退款原因（功能BUG、充值未到账、充值过多、未达到预期、账号注销、其他问题）
3. 填写详情描述
4. 填写需要售后的产品名
5. 付款订单号为选填项，如有请填写
6. 点击"确认提交"
7. **保存好系统返回的售后编号，用于查询进度**

### 查询处理进度

1. 点击"查询进度"导航
2. 输入售后编号（格式如：AS20240315XXXXXX）
3. 点击"查询"按钮
4. 查看处理状态和结果

## 🔐 后台管理指南

### 登录后台

- 访问 `admin/login.php`
- 默认账号: `admin`
- 默认密码: `admin123`
- **建议首次登录后立即修改密码**

### 后台功能模块

#### 1. 概览页面
- 查看统计数据（待处理、处理中、已处理、总订单）
- 查看最新提交的售后申请

#### 2. 订单管理
- 查看所有售后申请列表
- 按状态筛选订单
- 搜索订单编号/产品名/付款订单号
- 查看订单详情
- 处理订单（修改状态、填写处理结果）

**订单状态说明**:
- 待处理：新提交的申请，等待处理
- 处理中：客服人员正在处理
- 已完成：问题已解决
- 已拒绝：申请不符合条件被拒绝

#### 3. 系统设置
- **邮件通知设置**: 配置SMTP邮件服务器，用户提交申请时自动发送邮件通知管理员
  - 发信人邮箱：发送邮件的邮箱账号
  - 授权码：邮箱SMTP授权码
  - 收件人邮箱：接收通知的管理员邮箱
  - SMTP地址：邮件服务器地址（如smtp.qq.com）
  - 端口：SMTP端口（如465）
  - 邮件标题：通知邮件的标题
  - API密钥：邮件发送接口的密钥

- **提交限制设置**: 限制每个IP每天可提交的售后申请次数
  - 设置为0表示不限制
  - 建议设置为3-5次，防止恶意刷单

- **修改管理员密码**: 修改后台登录密码
- **查看系统信息**: 查看PHP版本、数据库状态等

#### 4. 关于系统
- 查看开发公司信息
- 访问官方网站
- 联系客服支持

## 📊 数据处理流程

```
用户提交申请 → 生成售后编号 → 状态:待处理
                          ↓
管理员查看订单 → 开始处理 → 状态:处理中
                          ↓
处理完成 → 填写结果 → 状态:已完成/已拒绝
                          ↓
用户查询进度 ← 显示状态和结果
```

## ⚙️ 常见问题

### Q1: 忘记后台密码怎么办？
**A**: 可以通过数据库直接修改密码：
```sql
UPDATE admin SET password = '$2y$10$92IXUNpkjO0rOQ5byMi.Ye4oKoEa3Ro9llC/.og/at2.uheWG/igi' WHERE username = 'admin';
```
修改后密码为: `password`

### Q2: 付款订单号可以不填吗？
**A**: 可以。付款订单号为选填项，建议有订单号时填写以便核实。

### Q3: 如何修改网站标题和版权信息？
**A**: 修改 `db.php` 文件中的 `getSystemInfo()` 函数。

### Q4: 数据库连接失败怎么办？
**A**: 检查以下几点：
- 数据库服务器是否正常运行
- 用户名和密码是否正确
- 数据库用户是否有足够权限
- PHP是否安装了PDO扩展

### Q5: 如何备份数据？
**A**: 可以通过phpMyAdmin或命令行导出数据库：
```bash
mysqldump -u root -p aftersales > backup.sql
```

### Q6: 邮件通知发送失败怎么办？
**A**: 请检查以下几点：
- 邮件配置中的所有参数是否正确填写
- SMTP地址和端口是否与发信邮箱匹配
- 发信邮箱是否已开启SMTP服务
- 授权码是否正确（注意不是登录密码）
- 服务器是否支持curl函数

### Q7: 用户提示"今日提交次数已达上限"怎么办？
**A**: 说明该IP今日提交次数已达到后台设置的限制。可以：
- 建议用户明天再提交
- 在后台系统设置中调高每日提交次数限制
- 设置为0可取消限制（不推荐）
